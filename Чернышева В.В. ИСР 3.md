# Инвариантная самостоятельная работа №3

## Задание 1. Юзабилити-тестирование

**Компонент образовательной среды:** Образовательная онлайн‑платформа Coursera

### Цели тестирования:
- Проверить, насколько удобно преподавателям и/или студентам находить курсы, регистрироваться, проходить обучение и использовать доступные функции платформы.
- Определить узкие места интерфейса и сложные для понимания функции.
- Получить рекомендации по улучшению пользовательского опыта (UX).

### Задачи для тестирования:
1. Зарегистрироваться на платформе и пройти авторизацию.
2. Найти и записаться на выбранный курс.
3. Просмотреть учебные материалы и пройти тесты внутри курса.
4. Просмотреть результаты выполнения тестов и прогресс по курсу.
5. Использовать дополнительные функции платформы (сертификаты, обсуждения, задания).

### Выбор участников:
- **Целевая аудитория:** студенты и преподаватели, пользователи онлайн-обучения.
- **Количество участников:** 6 студентов КЭО
- **Критерии:** опыт работы с электронными образовательными ресурсами, различный уровень технической подготовки.

### Подготовка материалов:
- **Скрипт тестирования:** инструкции для участников, описание задач.
- **Формы для сбора данных:** опросник после теста, таблица для фиксации проблем.
- **Техническая подготовка:** доступ к платформе Coursera, рабочая учетная запись, экранная запись (опционально).

### Выполнение задач:
- Участники выполняют заранее подготовленные задачи.
- Фиксируется:
  - Время выполнения каждой задачи
  - Ошибки и затруднения
  - Комментарии участников
- Ведущий тестирования записывает все затруднения, комментарии и положительные моменты.

### Классификация проблем:
- **Критические:** препятствуют выполнению задачи (например, невозможность пройти курс или сдать тест).
- **Значимые:** вызывают замедление или трудности, но задача выполнена (например, трудности с навигацией).
- **Незначительные:** мелкие неудобства, не влияющие на выполнение задач (например, непонятные элементы интерфейса).

### Метод оценки:
10‑балльная шкала удовлетворенности интерфейсом (1 – крайне неудовлетворен, 10 – полностью удовлетворен)

### Оценки участников:

| ID участника | Оценка (10-балльная шкала) |
|--------------|----------------------------|
| 1            | 6                          |
| 2            | 7                          |
| 3            | 8                          |
| 4            | 7                          |
| 5            | 6                          |
| 6            | 9                          |

- **Средняя оценка:**

$$
\bar{x} = \frac{6+7+8+7+6+9}{6} = \frac{43}{6} \approx 7.17
$$

- **Минимальная оценка:** 6  
- **Максимальная оценка:** 9  

**Распределение оценок:**

| Балл | Кол-во участников |
|------|-----------------|

### Основные выявленные недочёты:
- **Сложности с поддержкой:** медленная или неэффективная служба поддержки.
- **Технические и интерфейсные проблемы:**
  - ошибки при загрузке видео;
  - синхронизация прогресса между устройствами работает не всегда корректно;
  - интерфейс местами перегружен или неинтуитивен.
- **Качество аналитики и прогресса:** не всегда понятно, какой прогресс достигнут, как проходят задания.
- **Мобильная адаптация:** платформа не всегда удобно работает на мобильных устройствах.

**Интерпретация:**  
Наиболее проблемные области — поддержка и мобильная работа.  
Сильная сторона — качество курсов и контента.

## Задание 2. Юзабилити-тестирование с использованием шкалы "System Usability Scale" (SUS)

**IT-продукт:** Quizlet

### Цели тестирования:
- Проверить, насколько удобно студентам и преподавателям создавать, редактировать и использовать учебные карточки и наборы заданий на Quizlet.
- Определить узкие места интерфейса и сложные для понимания функции.
- Получить рекомендации по улучшению UX.

### Задачи для тестирования:
1. Зарегистрироваться на платформе и пройти авторизацию.
2. Создать новый набор карточек.
3. Добавить карточки разных типов (текст, изображение, аудио).
4. Использовать режимы обучения: Flashcards, Learn, Write, Test.
5. Просмотреть результаты выполнения тестов и прогресса.
6. Отредактировать существующий набор карточек.

### Выбор участников:
- **Целевая аудитория:** студенты и преподаватели, пользователи онлайн-образования.
- **Количество участников:** 6 студентов КЭО.
- **Критерии:** опыт работы с электронными образовательными ресурсами, различный уровень технической подготовки.

### Подготовка материалов:
- **Скрипт тестирования:** инструкции для участников, описание задач.
- **Формы для сбора данных:** таблица для фиксации ошибок и комментариев, опросник SUS.
- **Техническая подготовка:** рабочая учетная запись на Quizlet, доступ к интернету, экранная запись (опционально).

### Проведение тестирования:
- Участники выполняют заранее подготовленные задачи.
- Фиксируется:
  - Время выполнения каждой задачи
  - Ошибки и затруднения
  - Комментарии участников

### Оценка по шкале SUS:
- После выполнения всех задач участники заполняют опросник SUS (10 вопросов, 5-балльная шкала).
- **Расчет оценки SUS:**
  - Для каждого нечётного вопроса: $score = answer - 1$
  - Для каждого чётного вопроса: $score = 5 - answer$
  - Суммируем баллы по всем 10 вопросам и умножаем на 2.5 → итоговая оценка 0–100.

- **Интерпретация:** SUS ≥ 68 — средний уровень, выше — хороший UX, ниже — слабый UX.

### Классификация проблем:
- **Критические:** препятствуют выполнению основной задачи (например, невозможно создать карточки).
- **Значимые:** замедляют выполнение задачи, но решение возможно (например, трудности с поиском функций).
- **Незначительные:** мелкие неудобства интерфейса, не влияющие на результат (например, визуальные недочёты).

### Результаты:

| Участник | SUS |
|----------|-----|
| 1        | 75  |
| 2        | 80  |
| 3        | 70  |
| 4        | 85  |
| 5        | 65  |
| 6        | 78  |

- **Средняя оценка SUS:**

$$
\text{SUS}_{avg} = \frac{75 + 80 + 70 + 85 + 65 + 78}{6} = \frac{453}{6} \approx 75.5
$$

- **Интерпретация:** средняя оценка > 68 → интерфейс Quizlet оценивается как выше среднего, удобный для использования.

### Основные выявленные проблемы:
- Трудности с поиском определённых функций (например, добавление аудио).
- Ограниченные возможности настройки тестов и карточек для преподавателей.
- Не всегда очевидная навигация между режимами обучения.
- Мобильное приложение работает медленнее некоторых функций веб-версии.
