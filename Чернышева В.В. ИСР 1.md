## Задание 2. Создание диаграммы Ганта

**Выбранный компонент образовательной среды для проектной деятельности:** Автоматизированный конструктор проверочных работ  

### Постановка задач

#### 1. Анализ
1.1 Изучить существующие таксономии учебных целей (например, Блум)  
1.2 Разработать структуру задания с обязательными метаданными: тема, сложность, тип (мн. выбор/открытый), проверяемый навык, время на выполнение  
1.3 Спроектировать простую ML-модель (или правило) для рекомендации заданий. Например: “Если >30% класса ошиблось в задании на тему X, предложить 2 задания на закрепление этой темы”  

#### 2. Проектирование
2.1 **База данных:** Спроектировать схему для хранения банка заданий и результатов с возможностью быстрой агрегации (PostgreSQL + JSONB для хранения структурированных ответов)  
2.2 **Логика:** Алгоритм формирования вариантов с уникализацией (перемешать вопросы и варианты ответов)  
2.3 **Интерфейс:** Макет страницы, где учитель видит “диагностическую карту” класса и кнопку “Сгенерировать работу на ликвидацию пробелов”  

#### 3. Разработка
3.1 Реализовать сервис генерации вариантов. На вход: список тем, сложность, длительность. На выход: набор заданий  
3.2 Написать модуль экспорта работы в различные форматы (PDF для печати, JSON для импорта в СДО)  
3.3 Реализовать базовый анализ ответов: подсчет % выполнения по темам, построение графиков (используя библиотеку `pandas` на бэкенде и отправляя данные на фронтенд)  
3.4 Создать систему “типовых ошибок” – шаблонов, которые педагог может прикреплять к заданию для быстрой обратной связи ученику  

#### 4. Тестирование
4.1 **Функциональное:** проверить, что сгенерированные варианты соответствуют критериям (нет дубликатов, охвачены все запрошенные темы)  
4.2 **Юзабилити-тест с педагогами:** насколько интуитивно понятен процесс создания “умной” работы  
4.3 **Тестирование корректности анализа результатов** (сравнить с ручным подсчетом)
